Support Vector Machines¶

Support vector machines (SVMs) bear some similarity to logistic regression. They learn a particular linear decision boundary called the maximum margin hyperplane (mmh). In a 2-class, linearly separable problem, the mmh is the hyperplane maximizing the distance to the closest points of each class (these points are called support vectors).

Only the support vectors are relevant to defining the hyperplane. The other points are not, and this indicates that SVMs are resilient to overfitting (adding a new point far from the support vectors doesn't affect the system).

In scikit-learn, the package sklearn.svm defines several SVM implementations for classification and regression.

Creating a Classifier¶

svm.SVC, svm.NuSVC and svm.LinearSVC are classifier algorithms, the latter of these assuming a linear kernel.

Each is capable of performing multiclass classification. If there are $n$ distinct classes, LinearSVC generates $n$ distinct class models (one vs. rest). An exception to this is if there are only two classes; in that case, only one model is needed.

For SVC and NuSVC, a model is generated to distinguish pairs of classes (one vs one).

The wdbc data set has only two classes. For data sets with more classes, decision_function will return a sequence of values for each instance. The values are confidence scores for each class.

The dimensions of the returned matrix will depend on which decision function type is used.

  • svm.SVC and svm.NuSVC use a one-vs-one scheme in which $n(n-1)/2$ values are generated (pairs of classes are used to generate models).
  • LinearSVC uses a a one-vs-rest scheme, in which $n$ values (for $n$ classes) are generated.

Visualization¶

The below code (from Python Machine Learning, 2e, by Sebastian Raschka) defines a function which will graphically display both data points as well as decision boundaries between classes.

In [6]:
from matplotlib.colors import ListedColormap
import matplotlib.pyplot as plt


def plot_decision_regions(X, y, classifier, test_idx=None, resolution=0.02):

    # setup marker generator and color map
    markers = ('s', 'x', 'o', '^', 'v')
    colors = ('red', 'blue', 'lightgreen', 'gray', 'cyan')
    cmap = ListedColormap(colors[:len(np.unique(y))])

    # plot the decision surface
    x1_min, x1_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    x2_min, x2_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx1, xx2 = np.meshgrid(np.arange(x1_min, x1_max, resolution),
                           np.arange(x2_min, x2_max, resolution))
    Z = classifier.predict(np.array([xx1.ravel(), xx2.ravel()]).T)
    Z = Z.reshape(xx1.shape)
    plt.contourf(xx1, xx2, Z, alpha=0.3, cmap=cmap)
    plt.xlim(xx1.min(), xx1.max())
    plt.ylim(xx2.min(), xx2.max())

    for idx, cl in enumerate(np.unique(y)):
        plt.scatter(x=X[y == cl, 0], 
                    y=X[y == cl, 1],
                    alpha=0.8, 
                    c=colors[idx],
                    marker=markers[idx], 
                    label=cl, 
                    edgecolor='black')

    # highlight test samples
    if test_idx:
        # plot all samples
        X_test, y_test = X[test_idx, :], y[test_idx]

        plt.scatter(X_test[:, 0],
                    X_test[:, 1],
                    c='',
                    edgecolor='black',
                    alpha=1.0,
                    linewidth=1,
                    marker='o',
                    s=100, 
                    label='test set')

Using SVMs with non-linear problems¶

It is possible to develop classifiers for data sets that are not linearly separable. In these cases, a different "kernel" is used.

For SVM the possible values for kernel are:

  • linear
  • poly (polynomial)
  • rbf: (radial basis function)
  • sigmoid

For some, other parameters must be specified (e.g., gamma).

Below is a scatter plot of data that generates random points based on the XOR function (which is not linearly separable).
